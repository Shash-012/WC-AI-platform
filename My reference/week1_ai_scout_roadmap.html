[ WORLD CUP AI SCOUT — PROJECT 1 ]

Week 1 Build Roadmap

RAG pipeline from zero to working in a terminal script. No frontend. No Docker. Just the intelligence layer.

▶ End goal: chatbot answers "Who plays in Group A?" correctly from a Python script. That's the only definition of done.
01
// priority 1 — learn first
LangChain + RAG Architecture
Foundation
LangChain
The glue between your data, embeddings, and LLM. Learn the Document Loader → Text Splitter → Embeddings → VectorStore → RetrievalQA chain. That one chain is 90% of what you're building.
Start → python.langchain.com/docs/get_started/quickstart
Video → youtube: "LangChain RAG tutorial" by Sam Witteveen
~4 hrs to get comfortable
RAG Concept
Retrieval-Augmented Generation = "search your own docs, then ask the LLM." Read the original RAG paper abstract (don't go deep), then watch one explainer video. The concept clicks fast — the implementation is where the time goes.
Read → arxiv.org/abs/2005.11401 (abstract only)
Video → youtube: "What is RAG?" by IBM Technology
~1 hr to understand the mental model
02
// priority 2 — learn alongside langchain
FAISS + Embeddings
Vector Store
FAISS
Facebook's similarity search library. You won't use it directly much — LangChain wraps it. What matters: understand that it stores vectors (number arrays) and finds the closest match. Don't go deep on the algorithm. Just know how to call FAISS.from_documents() and .save_local().
Docs → python.langchain.com/docs/integrations/vectorstores/faiss
~1 hr (mostly just use it via LangChain)
Embeddings Concept
Text → number arrays that capture meaning. Similar text = similar numbers. You'll use all-MiniLM-L6-v2 from HuggingFace as your model — it runs locally, no API key needed. Start with OpenAI embeddings first to test, then swap to HuggingFace in Week 2.
Video → youtube: "Word Embeddings Explained" by Andrej Karpathy
HF → huggingface.co/sentence-transformers/all-MiniLM-L6-v2
~2 hrs concept + hands-on
03
// priority 3 — data before models
Data Collection + Chunking Strategy
Data Layer
API-Football
Free tier gives you groups, fixtures, team squads, standings. Pull data as JSON, convert to readable text chunks. Your FUTZONE project already covers the API pattern — same idea.
API → api-football.com (RapidAPI, free tier)
~2 hrs data pulling
Text Chunking
Split your data into small, meaningful chunks before embedding. LangChain's RecursiveCharacterTextSplitter is the default. Aim for ~500 token chunks with 50-token overlap.
Docs → python.langchain.com/docs/modules/data_connection/document_transformers
~1 hr to implement
Static Fallback Data
Don't block on the API. Manually write 20-30 text chunks as .txt files covering groups, key players, host cities. Test the RAG pipeline with this first, replace with live API data after.
Source → fifa.com/worldcup/groups
~2 hrs writing chunks
04
// priority 4 — set up early, use throughout
LLM API + Prompt Engineering
Intelligence Layer
Gemini API (recommended)
Generous free tier — 1500 requests/day. No credit card for dev. Use gemini-2.0-flash. LangChain has a ChatGoogleGenerativeAI wrapper — plug-and-play with your RAG chain. OpenAI works too if you already have a key.
Start → ai.google.dev (get API key free)
LangChain → python.langchain.com/docs/integrations/llms/google_ai
~1 hr setup, 30 min first call
Prompt Engineering
Your system prompt tells the LLM to answer only from retrieved context. Learn the "context injection" pattern: retrieved chunks go into the prompt, LLM is told not to make up facts outside them. This is what makes RAG actually accurate vs hallucinating.
Guide → learnprompting.org/docs/basics/introduction
Deep dive → promptingguide.ai — "RAG Prompting" section
~2 hrs reading + iterating prompts
[ END-OF-WEEK-1 CHECKPOINT ]
Working RAG chatbot in a Python terminal script
Run python chat.py, type a question, get a grounded answer. That's the week 1 bar. Nothing else ships yet.
FAISS vector index built from your World Cup text chunks
LangChain RetrievalQA chain wired to Gemini or OpenAI
Answers questions about groups, fixtures, and players from your data
System prompt stops hallucination — answers are grounded in retrieved docs only
// scope rule
If you feel the urge to build the Flask API or start the React frontend this week — don't. The pipeline isn't real until it answers questions accurately in a script. Ship that first.
// learning order
Read just enough to write the next 10 lines of code. Don't finish entire courses before touching the project. LangChain quickstart → copy a RAG example → modify it is the fastest path.
